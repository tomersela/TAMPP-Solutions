### Model

Treat the 15% / 85% split as Amdahl’s serial / parallel fractions of work measured on the baseline uniprocessor run: the sequential chunk stays on one core, while the parallel chunk is spread over $N$ cores.

**Operation mix.** Let $p_{\mathrm{mem}} = 0.2$ and $p_{\mathrm{other}} = 1 - p_{\mathrm{mem}} = 0.8$. A non-memory operation costs $1$.

**Parallel cache behavior.** On a parallel memory access, the exercise sets $\mathrm{CacheMiss}(N) = \dfrac{N}{N + 10}$. That value is the probability of a miss.<br/>
The hit probability is $1$ minus that, since each access is either a hit or a miss: $1 - \mathrm{CacheMiss}(N) = \dfrac{10}{N + 10}$, and $\dfrac{10}{N + 10} + \dfrac{N}{N + 10} = 1$. Hit cost is $1$, miss cost is $3N + 11$. Averaging hit and miss gives the expected cost of one parallel memory access in the next display; a random parallel operation costs $p_{\mathrm{mem}}$ times that plus $p_{\mathrm{other}} \cdot 1$.

For one operation drawn at random from the 15% serial portion (same 20% memory / 80% non-memory mix as everywhere, but sequential memory costs $14$ and non-memory costs $1$):

$$
\text{cost} = 0.2 \cdot 14 + 0.8 \cdot 1 = 3.6.
$$

Focus on the parallel portion. The exercise gives a miss *rate* $\mathrm{CacheMiss}(N) = \dfrac{N}{N + 10}$:<br/>
Each time the parallel code performs a memory access, it misses the cache with that probability and hits with probability $1 - \mathrm{CacheMiss}(N) = \dfrac{10}{N + 10}$. A hit costs $1$ unit (the cheap “cache and other local” case). A miss costs $3N + 11$ units (the expensive parallel main-memory path).

So, **for one memory access in the parallel part only** (not yet mixing in the 80% non-memory ops), the expected cost is a weighted average of those two outcomes:

$$
\underbrace{\frac{10}{N+10}}_{\text{hit}} \cdot 1 + \underbrace{\frac{N}{N+10}}_{\text{miss}} \cdot (3N + 11) = \frac{10 + N(3N + 11)}{N + 10}.
$$

**One parallel operation overall** is a memory access with probability $0.2$ and a non-memory operation with probability $0.8$ (still cost $1$). Let $C_{\mathrm{par}}(N)$ be the expected cost of one such operation on the parallel (85%) code path when you use $N$ processors. Then:

$$
C_{\mathrm{par}}(N) = 0.2 \cdot \frac{10 + N(3N + 11)}{N + 10} + 0.8.
$$

Execution time as a function of $N$ (up to an irrelevant global scale) is therefore:

$$
T(N) =
\underbrace{0.15}_{\text{serial (15\%)}}
\cdot
\underbrace{3.6}_{\text{serial op cost}}
+
\underbrace{0.85}_{\text{parallel (85\%)}}
\cdot
\underbrace{\frac{C_{\mathrm{par}}(N)}{N}}_{\text{parallel op cost / N cores}}
$$

The first term is fixed. The second balances lower latency from parallelism ($1/N$) against worse parallel memory behavior as $N$ grows.

### Discrete optimum

**One minimum.** The first term of $T(N)$ is constant in $N$, so minimizing $T$ is the same as minimizing

$$
g(N) = \frac{C_{\mathrm{par}}(N)}{N}.
$$

Expand $C_{\mathrm{par}}(N)$ (algebra from the formula above): for $N > 0$,

$$
C_{\mathrm{par}}(N) = \frac{3}{5}N - 3 + \frac{40}{N + 10},
$$

so

$$
g(N) = \frac{3}{5} - \frac{3}{N} + \frac{40}{N(N + 10)}.
$$

Differentiate on $(0,\infty)$:

$$
g'(N) = \frac{3}{N^2} - \frac{40(2N + 10)}{N^2(N + 10)^2} = \frac{1}{N^2}\left(3 - \frac{40(2N + 10)}{(N + 10)^2}\right).
$$

Setting the bracket to zero gives $3(N + 10)^2 = 40(2N + 10)$, i.e. $3N^2 - 20N - 100 = 0$. The positive root is $N = 10$; there are no others. For $0 < N < 10$ one has $g'(N) < 0$, and for $N > 10$, $g'(N) > 0$, so $g$ has a **single** global minimum on $(0,\infty)$ at $N = 10$. The same holds for $T(N) = 0.54 + 0.85\,g(N)$. On the integers $N \geq 1$, the minimizer is therefore $N = 10$ (strictly better than $9$ and $11$).

Sample values near that minimum:

| $N$ | $T(N)$ (relative) |
|----:|------------------:|
| 8   | $\approx 0.96736$ |
| 9   | $\approx 0.96550$ |
| 10  | $0.965$   |
| 11  | $\approx 0.96537$ |
