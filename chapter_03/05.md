No. Dropping L2 is not the same as sequential consistency.

**L1 without L2** - You only require some extension $H'$ and a legal sequential $S$ with $\mathit{complete}(H')$ equivalent to $S$. That says nothing about how operations from the same thread must be ordered in $S$.

**Sequential consistency** - Also requires that, for each thread, program order is preserved in $S$.

**What L2 is** - $m_0 \rightarrow_H m_1$ (typically: $m_0$’s response before $m_1$’s invocation) must hold in $S$ too. That pins real-time order between calls; it is not the same statement as SC’s per-thread program order. Removing L2 drops the former; it does not add the latter.

So L1-only and SC are different notions (some blurbs equate “SC” with “linearizability minus real time”; that skips SC’s explicit program-order clause. The book’s definitions are what matter here.)
