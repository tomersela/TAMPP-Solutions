1. **Deadlock-free**  
   For every infinite history, infinitely many method calls complete.  
   This is system-wide progress: the execution never reaches a state where no further calls ever complete. Some threads may still starve.

2. **None of these**  
   For every finite history, there exists an infinite extension.  
   This only says executions can continue forever. It says nothing about completed method calls or progress.

3. **None of these**  
   For every infinite history, every thread takes infinitely many steps.  
   This is a scheduler/fairness property, not an object progress property.

4. **Starvation-free**  
   Every thread that takes infinitely many steps completes infinitely many method calls.  
   This is per-thread progress: if a thread keeps running, it keeps completing calls.

5. **Obstruction-free**  
   After any finite history, if only thread $i$ runs, it can complete infinitely many method calls.  
   This is exactly progress in isolation.

6. **None of these**  
   After any finite history, there exists an infinite extension where every thread completes infinitely many method calls.  
   This is only an existential “good schedule” property, not one of the standard progress conditions.
