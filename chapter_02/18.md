
Initial valid state:
A = 00, B = 01, C = 10

- A and B perform concurrent scan() operations and both see the same old state
- Based on that same snapshot, both choose 11
- Now state is: A = 11, B = 11, C = 10
- Then A performs another label() and moves to 12

Final state: A = 12, B = 11, C = 10

These three labels lie in the same T^2 copy, so they form a cycle rather than a total order. Therefore T^3 does not work as a concurrent timestamp system for 3 threads.
