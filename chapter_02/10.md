We need a doorway because FCFS needs a precise point at which a thread is considered to have officially joined the race for the lock, and the first instruction of lock() is not enough for that.


**Read of a separate location** - 
This leaves no shared evidence visible to other threads. So even if A executes that read before B, the algorithm has no way to use that fact to enforce FCFS.

**Read of the same location** - Same problem. Reads do not change memory, so the order of those first reads is not recorded anywhere. Later threads cannot tell who was first.

**Write to a separate location** - After both writes happen, shared memory shows that both writes occurred, but it still does not reveal which one happened first. So first-instruction order is not recoverable, and FCFS cannot be enforced from it.

**Write to the same location** - The later write can overwrite the earlier one, so evidence of who wrote first may be destroyed completely. Again, the algorithm cannot reliably know the first-instruction order.

So the doorway is needed because it gives a bounded, meaningful arrival point: "A finished its doorway before B started its doorway" is something the algorithm can base FCFS on, unlike "A executed the first instruction first."
