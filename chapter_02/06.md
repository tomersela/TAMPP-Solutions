## Intuition

The tree lock is built from 2-thread Peterson locks arranged in a binary tree.

A thread acquires the lock by climbing from its leaf to the root, taking one Peterson lock at each level. Since Peterson already guarantees mutual exclusion, deadlock-freedom, and starvation-freedom for 2 contenders, the main question is whether these properties survive composition in the tree.

The key observations are:

* To enter the critical section, a thread must win the root Peterson lock, so mutual exclusion should follow from the root alone.
* Threads always move upward in the tree, never in circles, so deadlock should not happen.
* Once a thread reaches some node, it is the only contender from its subtree at that node, because it already holds all lower locks on its path. So at each node it faces only one opposing side, and Peterson's bounded-waiting property applies.
* Since the path from a leaf to the root has exactly $\log_2 n$ levels, the total delay is bounded by the sum of the delays at each level.


## 1. Mutual exclusion

### Lemma.

The tree-lock algorithm satisfies mutual exclusion.

### Proof.

Suppose not. Then two distinct threads A and B are simultaneously in the critical section.

Any thread that enters the critical section must first acquire every Peterson lock on its path from its leaf to the root. In particular, both A and B must have acquired the Peterson lock at the root.

But the root is just a 2-thread Peterson lock, and Peterson's algorithm satisfies mutual exclusion. So at most one thread can hold that lock at a time.

This is a contradiction. Therefore the tree-lock satisfies mutual exclusion.


## 2. Freedom from deadlock

### Lemma.

The tree-lock algorithm is deadlock-free.

### Proof.

Every thread acquires locks in the same direction: from its leaf toward the root. No thread ever goes back downward while acquiring.

Suppose some set of threads is trying to acquire the tree-lock. At any internal node where two threads meet, they compete using a 2-thread Peterson lock. Since Peterson is deadlock-free, one of the two eventually wins that node and moves upward.

Because the tree has finite height, some thread must continue making progress level by level until it reaches the root. Once it acquires the root lock, it acquires the tree-lock.

So it is impossible for all trying threads to be stuck forever. Therefore the tree-lock is deadlock-free.


## 3. Freedom from starvation

### Lemma.

The tree-lock algorithm is starvation-free.

### Proof.

Consider a thread $A$ trying to acquire the tree-lock. Its path from leaf to root contains exactly $h = \log_2 n$ Peterson locks.

Fix one node $v$ on this path. Once $A$ reaches $v$, it already holds all locks below $v$ on its path. Therefore no other thread from the same subtree can compete with $A$ at $v$. So at node $v$, $A$ is the unique contender from its side.

The only competition comes from the sibling subtree, which appears as the other side of the 2-thread Peterson lock at $v$. Peterson is starvation-free, and in fact has 1-bounded waiting: once one side is waiting, the other side can overtake it at most once before it must get the lock.

Therefore, at each node on its path, thread $A$ can be delayed only a finite amount before acquiring that node.

Since there are only $h = \log_2 n$ nodes on the path, $A$ eventually acquires them all, including the root. Hence $A$ eventually acquires the tree-lock.

Therefore the tree-lock is starvation-free.


## 4. Upper bound on overtaking

### Corollary.

There is an upper bound on the number of times the tree-lock can be acquired and released between the time a thread starts acquiring it and when it succeeds, and that bound is $\log_2 n$.

### Proof.

A thread must pass through exactly one Peterson lock per level, and the tree has $\log_2 n$ levels.

At each level, Peterson's bounded-waiting property implies that the competing side can overtake the thread at most once before the thread acquires that node.

Therefore, over the whole path, the number of times other threads can acquire and release the tree-lock before the waiting thread succeeds is at most $\log_2 n$.
