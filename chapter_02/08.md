The scientists are wrong. The wrapper is not correct.


Let A and B be two threads. Initial state:

* `x` arbitrary
* `y = -1`
* base `lock` is free

Execution:

1. A: `x = A`
2. A: reads `y == -1` at line 8, so it leaves the loop
3. B: `x = B`
4. B: reads `y == -1` at line 8, so it also leaves the loop
5. B: `y = B`
6. B: reads `x == B` at line 10, so it does not call `lock.lock()`
7. B enters the critical section
8. A: `y = A`
9. A: reads `x != A` at line 10, since now `x == B`
10. A calls `lock.lock()`
11. Since B never acquired the base lock, A finds it free and acquires it
12. A enters the critical section

Now both A and B are in the critical section at the same time:

* B got in through the fast path
* A got in through the slow path

So mutual exclusion is violated.
