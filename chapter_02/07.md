The original $n$-thread Filter lock uses $n - 1$ levels so that at each level at least one contender is left behind.

Therefore at most one thread reaches the critical section. To obtain an $\ell$-exclusion algorithm, use only $(n - \ell)$ levels. By the same argument, after passing all levels, at most $\ell$ threads can remain, so the algorithm satisfies $\ell$-exclusion. The progress argument is the natural generalization of Filter: if fewer than $\ell$ threads are in the critical section, then some waiting thread eventually advances and enters, so the algorithm satisfies $\ell$-starvation-freedom.

```java
class LExclusion implements Lock {
    private final int n;
    private final int levelCount;
    private final int[] level;
    private final int[] victim;

    public LExclusion(int n, int l) {
        this.n = n;
        this.levelCount = n - l;
        this.level = new int[n];
        this.victim = new int[levelCount + 1]; // indices 1 .. levelCount
        for (int i = 0; i < n; i++) {
            level[i] = 0;
        }
    }

    public void lock() {
        int me = ThreadID.get();
        for (int i = 1; i <= levelCount; i++) {
            level[me] = i;
            victim[i] = me;
            while (otherWaitingAtLevel(me, i)) { }
        }
    }

    private boolean otherWaitingAtLevel(int me, int i) {
        for (int k = 0; k < n; k++) {
            if (k != me && level[k] >= i && victim[i] == me) {
                return true;
            }
        }
        return false;
    }

    public void unlock() {
        level[ThreadID.get()] = 0;
    }
}
```
