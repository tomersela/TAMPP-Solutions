### Mutual exclusion

Suppose \(i < j\) and both are in the critical section. Thread \(j\) finished the first phase only after some pass of the `for (int j = 0; j < i; j++)` loop where every time it read `flag[k]` for \(k < j\) (in particular for \(k = i\)) the value was `false`. Thread \(i\) had already set `flag[i] = true` for the successful attempt that led to the CS and does not clear it before `unlock()`, so while \(i\) is in the CS, `flag[i]` stays `true`. The read by \(j\) cannot have seen `false`. Contradiction. The case \(j < i\) is symmetric.

### Deadlock-free

Assume sequential consistency and fair scheduling of each wait loop (each spinner eventually sees concurrent writes to the flag it reads).

Let \(m\) be the smallest id in `lock()`. It completes phase 1.

In phase 2, \(m\) waits only on `flag[k]` for \(k > m\). If \(k\) is still in phase 1, it will still scan \(j = m\) in its inner `for` while `flag[m]` is true and will set `flag[k] = false` (lines 12–13), so that wait of \(m\) cannot hang on such a \(k\) forever. If \(k\) has finished phase 1, it does not wait on \(m\). Phase-2 edges “\(i\) waits on \(k\)” all have \(i < k\), so there is no cycle. Some thread therefore finishes all its phase-2 spins (when the higher flags it waits on are cleared by backoff or `unlock()`), reaches the CS, and `unlock()` advances the others. Hence no fair execution deadlocks the whole set.
