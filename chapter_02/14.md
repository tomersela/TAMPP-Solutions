The original n-thread Filter lock uses n - 1 levels so that at each level at least one contender is left behind.

Therefore at most one thread reaches the critical section. To obtain an ℓ-exclusion algorithm, use only $(n - ℓ)$ levels. By the same argument, after passing all levels, at most $ℓ$ threads can remain, so the algorithm satisfies ℓ-exclusion. The progress argument is the natural generalization of Filter: if fewer than $ℓ$ threads are in the critical section, then some waiting thread eventually advances and enters, so the algorithm satisfies ℓ-starvation-freedom.


```java
class LExclusion {
    int levelCount;
    int[] level;
    int[] victim;

    public LExclusion(int n, int l) {
        levelCount = n - l;
        level = new int[n];
        victim = new int[levelCount + 1]; // use 1..levelCount
        for (int i = 0; i < n; i++) {
            level[i] = 0;
        }
    }

    public void lock() {
        int me = ThreadID.get();
        for (int i = 1; i <= levelCount; i++) {
            level[me] = i;
            victim[i] = me;
            while ((∃k != me) (level[k] >= i && victim[i] == me)) {}
        }
    }

    public void unlock() {
        int me = ThreadID.get();
        level[me] = 0;
    }
}
```
